# Code Peek v0.2 - Dual Mode Redesign

## 1. Project Context
Code Peek is a Chrome extension that extracts CSS, colors, typography, assets, and page info. Currently has 5 tabs (Overview, Colors, Typography, Assets, Inspect) plus basic toggles (dark mode, continuous inspect). Data extraction works well but persistence and export features are limited.

## 2. Goals for v0.2
- Introduce **Designer Mode** and **Developer Mode** with optimized tool sets.
- Add persistent settings (dark mode, continuous inspect, active tab, color subtab, mode, tab per mode).
- Implement full-page screenshot (stitched).
- Implement element export (HTML + computed CSS inlined).
- Implement full page export (standalone HTML with all assets inlined).
- Add Designer tools: palette generator, contrast checker, typography scale, ruler overlay.
- Add Developer tools: tech stack detection, copy CSS, CSS variables inspector, accessibility audit, performance hints.
- Smooth instant mode switching with tab memory per mode.

## 3. Modes & Architecture

### Mode Selection
- Header badge `[DESIGN]` or `[DEV]` + dropdown in menu.
- Persisted setting: `mode` ("designer" | "developer").
- Instant switch (no reload) restores the last active tab for that mode (`designerActiveTab`, `developerActiveTab`).

### Tab Bar (Bottom Navigation)
Dynamically renders 5 tabs based on current mode.

**Designer Mode tabs:**
1. `overview` (shared)
2. `colors` (enhanced)
3. `typography` (enhanced)
4. `rulers` (new)
5. `inspect` (shared)

**Developer Mode tabs:**
1. `overview` (shared)
2. `tech-stack` (new)
3. `code-snippets` (new)
4. `audit` (new)
5. `inspect` (shared)

**Shared:**
- `overview`: Page stats (stylesheets, load time, basic metrics).
- `inspect`: Element picker + detail panel with export actions.

### Header Actions
- Mode badge + dropdown (menu)
- "Viewport Screenshot" button (existing)
- "Full Page Screenshot" button (new)
- Reload button
- Menu button (utility menu with dark mode, continuous inspect)

## 4. Data Extraction Enhancements

Content script (`src/content/bundle.js`) will now provide:

| Feature | Data |
|---------|------|
| Tech Detection | Framework library (React, Vue, Angular, etc.), CSS framework (Tailwind, Bootstrap), build tool (Webpack, Vite), confidence score |
| Performance | Total asset size, render-blocking resources, oversized images (>100KB), unused CSS estimate |
| Accessibility | Contrast failures, missing alt attributes, form labels, heading hierarchy issues |
| Full HTML | `document.documentElement.outerHTML` |
| Computed CSS for element | `window.getComputedStyle(element)` for every CSS property |
| Images | All `<img>` + CSS `background-image` URLs with natural/widtheight |
| Fonts | `document.fonts` + CSS `@font-face` src URLs |
| CSS stylesheets | All `<link rel="stylesheet">` and `<style>` content |

These are requested on-demand via messaging.

## 5. Tool Specifications

### Designer Tools

#### Colors Tab (enhanced)
- Existing color list (name, hex, usage count).
- **Palette Generator**:
  - For each significant color (usage > threshold), generate harmony schemes:
    - Complementary (180°)
    - Analogous (30° shift)
    - Triadic (120° spacing)
    - Tetradic (90° spacing)
  - Show accessibility: for each foreground/background pair compute WCAG AA/AAA.
  - UI: grid of generated colors with lock toggle; "Generate New" button.
  - Export: copy CSS variables, SCSS map, or JSON.
- **Contrast Checker**: pick two colors (from list or custom) and see ratio + grade (Fail/A/AA/AAA).

#### Typography Tab (enhanced)
- List all font families with weight and sample text.
- **Scale Visualization**:
  - Detect base font-size from body.
  - Show typical modular scale (1.25 or golden ratio) with generated heading sizes (h1-h6).
  - Display line-height and letter-spacing trends.
  - Export as CSS variables.

#### Rulers Tab (new)
- Full-page overlay from content script:
  - **Guides**: Drag from edges to create horizontal/vertical lines. Persist per session (optionally per domain).
  - **Hover tooltip**: mousemove shows element dimensions (w×h) and distances to viewport edges (top, left, right, bottom).
  - **Measurement box**: click-drag to create temporary box showing selection size.
  - Controls: "Clear All" button, toggle guide snap to pixels.
- Tools integrated into one tab with toolbar.

### Developer Tools

#### Tech Stack Tab (new)
- Detect:
  - JS framework: React (`React`/`ReactDOM`), Vue (`Vue`), Angular (`ng`), Svelte, etc.
  - CSS framework: Tailwind (class patterns), Bootstrap (`.btn`, `.container`), Bulma, etc.
  - Build: Webpack (`webpackJsonp`), Vite (`vite`), Create React App, Next.js (`__next`).
  - Metas: generator tag, powered-by header.
- Display with confidence bar and evidence list (e.g., detected classes, globals).
- "Copy Report" button.

#### Code Snippets Tab (new)
- When element selected:
  - "Copy Computed CSS" (formatted, only declared properties, exclude browser defaults? maybe include).
  - "Copy HTML" (cleaned, minimal).
  - "Copy as Vue SFC" (single-file component with `<template>`, `<script>`, `<style scoped>`).
  - "Copy as React JSX" (component with `className`).
- "Export Full Site" → triggers full page HTML export (see below).

#### Audit Tab (new)
- **Accessibility**:
  - Color contrast fails (using same algorithm as designer tab).
  - Images missing `alt`.
  - Form inputs without associated labels.
  - Heading order issues (skipped levels).
- **Performance**:
  - Total uncompressed asset size estimate.
  - Images > 100KB flagged.
  - Render-blocking CSS/JS (scripts with no `async/defer`).
  - Unused CSS ballpark (heuristic: compare used selectors vs. total).
- Present as list with links (if possible) to elements; overall grades.

## 6. Export Engine

### Full Page Export (Standalone HTML)
User clicks "Export Full Page" → side panel sends `EXPORT_FULL_PAGE`.
Background (service worker) performs:

1. Get full HTML outerHTML.
2. Create a `<style>` block that concatenates:
   - All in-page `<style>` text.
   - Fetched content of all external stylesheets (as text), preserving order.
3. Rewrite CSS `url()` references:
   - For each URL, fetch as blob → base64 data URI.
   - Handle images, fonts (@font-face src).
4. Rewrite HTML `src`, `href` attributes for images, media, iframes:
   - Fetch each external resource and convert to data URI (images, audio, video).
   - For fonts, include in CSS step.
5. Inline critical CSS? Already inlined all.
6. Generate Blob → create download URL → `chrome.downloads.download`.
7. Stream progress (0-100) with textual phases:
   - "Fetching stylesheets..."
   - "Fetching images..."
   - "Converting assets to data URIs..."
   - "Generating file..."
8. Support cancellation via `chrome.runtime.onMessage` with `CANCEL_EXPORT` token.

**Implementation**: Use Web Worker in background? Service worker can't create dedicated worker, but can use async loops with `await`. Keep memory usage reasonable by batching.

### Single Element Export
User selects element → clicks "Export Element". Simpler:
- Get element's `outerHTML`.
- Get computed style for that element (and maybe its children).
- Create a style block: `element{ /* computed styles inlined */ }`.
- Apply inline styles to the element itself (set attribute `style="..."`).
- Wrap in minimal `<html><body>`.
- Download as `element.html`.

No need for massive asset inlining unless element contains images; could inline those too, but keep simple for now.

### Full Page Screenshot Stitching
Already partially implemented (captureFullScreenshot). Need to verify it scrolls and captures full height by repeated `chrome.tabs.captureVisibleTab` with scroll offsets, then stitch using canvas. If broken, fix.

## 7. Storage Schema

```javascript
{
  "darkMode": boolean,
  "continuousInspect": boolean,
  "mode": "designer" | "developer",
  "designerActiveTab": "overview" | "colors" | "typography" | "rulers" | "inspect",
  "developerActiveTab": "overview" | "tech-stack" | "code-snippets" | "audit" | "inspect",
  "colorSubtab": "all" | "categories",
  "rulers": { // optional, per-domain
    "guides": [{id, x, y, orientation}],
    "viewport": {width, height}
  }
}
```

Defaults:
- `mode`: "designer"
- `designerActiveTab`: "overview"
- `developerActiveTab`: "overview"
- `colorSubtab`: "all"

## 8. UI/UX Flow

1. **On startup**: Load storage; set mode badge; render tabs for mode; restore last active tab; load content for that tab.
2. **Switch mode**: Dropdown changes `mode`; save; hide current tab content; render new tab bar; activate saved `*ActiveTab` for new mode; load content.
3. **Tab click**: Update `activeTab` for current mode; save; switch content.
4. **Rulers**: Content script injects overlay based on message `ENABLE_RULERS`; draws UI; sends back measurement data on hover; guides stored in storage (volatile).
5. **Exports**: Show modal with progress bar and Cancel button. Background sends `PROGRESS` updates. Cancel clears any temp blobs and aborts.
6. **Screenshots**: Similar progress UI for full page stitch.

## 10. Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Full page export huge (MBs) | Show warning for large sites; allow cancel; stream in chunks |
| CORS blocks fetching external assets (images, fonts, CSS) from content script | Use background fetch with host permissions; use `fetch` from background (has webRequest?) Actually we can fetch via `fetch` in background with `host_permissions` covering all URLs; content script sends message to background to fetch URLs and return blobs. Background has host permissions due to manifest's `host_permissions`. |
| Canvas stitching memory for very tall pages | Capture in chunks; stitch incrementally; limit max height? |
| Tech detection false positives | Use conservative heuristics; show confidence |
| Computed CSS size huge | Only include necessary properties (standard set) for single element export |
| Web Workers in extension context | Side panel can create worker from `blob:` URL or we just keep main thread busy but show progress bar |

## 11. Success Criteria

- Both modes load within 2 seconds of opening side panel.
- Mode switch is instantaneous (<200ms).
- All tools functional on at least 80% of tested sites.
- Full page export produces a file that loads and looks identical (images, fonts, styles) when opened locally.
- No console errors in side panel on typical sites (except expected chrome:// errors).
- Memory usage < 500MB during export; cleanup after cancel.
- Auto-release GitHub Action creates tagged release with build artifacts.
